Compilando com o ShaderCompiler
por André "andrew1b" Santee

Shadercompiler é um pequeno programa freeware que possui seu código fonte aberto e pode ser usado tanto para estudo como para incrementações. O ShaderCompiler facilita muito a vida do programador 3D que utiliza shaders, poupando-o da necessidade de instalar inúmeros plugins no compilador ou apelar para programas que utilizam linha de código para compilar.

O ShaderCompiler possui um padrão de organização próprio no arquivo, que pode armazenar nome e informações sobre o shader além do binário próprio binário compilado. Basta uma função (contida no ShaderCompiler.h) para carregar o buffer com o shader e usá-lo para criar o Vertex ou Pixel shader posteriormente com as funções padrão do DirectX.

Como usar

O ShaderCompiler pode compilar vertex e pixel shaders do DirectX 9 (do vs_1_0 e ps_1_0 a ps_3_0 e vs_3_0).
Para compilar um shader basta executar o programa. Uma "open box" se abrirá, selecione o código do Pixel ou Vertex Shader e clique em "Abrir". Caso o shader tenha sido escrito corretamente, ele será compilado e uma "save box" surgirá para que você selecione o diretório de destino onde será salvo o arquivo binário do shader compilado.

Erros de compilação

Caso o compilador encontre algum erro de sintaxe no código do shader, um alerta indicará onde o erro foi encontrado e uma sugestão de o que pode estar causando o problema.

Informações técnicas sobre o shader

No arquivo ShaderInfo.txt, localizado no mesmo diretório onde se encontra o ShaderCompiler.exe, podem ser inseridas as informações sobre o shader.
Na segunda linha pode ser colocado um nome para o shader, por exemplo: "Directional phong shader", ou outro nome qualquer (máximo 64 caracteres).
Na terceira linha uma breve descrição sobre o que este shader faz, exemplo: "este shader calcula a atenuação de uma luz direcional e define a cor specular com base em phong shading" (máximo de 128 caracteres).
Na quarta e última linha pode ser inserido um texto mais detalhado de como foi feito o shader, cálculos e expressões utilizadas e também informações sobre os registradores de input e constantes utilizadas, o máximo de caracteres nesta string é de 512.

Para inserir o caractere de RETORNO (enter) nos textos de informações sobre o shader utilize a constante '\n', que será interpretada como a tecla de retorno padrão.
As strings contidas no arquivo ShaderInfo.txt serão anexadas ao arquivo onde será armazenado o shader compilado, porém são opcionais.

Lendo o shader

Ler o shader gerado pelo ShaderCompiler é muito fácil, basta utilizar a função scLoadShaderFromFile, no arquivo ShaderCompiler.h, veja o exemplo:

SC_SHADER_BIN scShaderBin;
scLoadShaderFromFile("bumpreflections.bin", &scShaderBin);

A estrutura SC_SHADER_BIN consiste em:

struct SC_SHADER_BIN
{
    void Release(); // função que limpa o buffer byBuffer
    SC_SHADER_FILE scShaderFile; // cabeçalho do arquivo
                                 // aqui temos as strings
                                 // sobre o shader
    unsigned char *byBuffer; // neste buffer será
                             // alocado o shader
};

A estrutura SC_SHADER_FILE:

struct SC_SHADER_FILE
{
    char szShaderName[64];     // nome do shader
    char szShortDesc[128];     // descrição breve
    char szLongDesc[512];      // descrição longa
    unsigned long nBufferSize; // tamanho do buffer
};

Após chamada a função, scShaderBin::byBuffer podem ser utilizados os métodos IDirect3DDevice9::CreateVertexShader ou CreatePixelShader para criar o shader.

Exemplo de shader

Aqui vai um pequeno shader pra quem quer algo para testar o mini-compilador:

#define CV_WORLDVIEWPROJ_0 0 ; world*view*proj transposta
#define CV_WORLDVIEWPROJ_1 1
#define CV_WORLDVIEWPROJ_2 2
#define CV_WORLDVIEWPROJ_3 3

#define CV_BLEND_FACTORS   4 ; t, 1.0-t, 0.0, 1.0
#define CV_CONSTANTS       5 ; 0.0, 0.5, 1.0,-1.0

#define CV_LIGHT0          6 ; R, G, B, Alcance
#define CV_LIGHTPOS_0      7 ; Posição da luz

#define CV_WORLD_0         8 ; world transposta
#define CV_WORLD_1         9
#define CV_WORLD_2        10
#define CV_WORLD_3        11

#define CV_AMBIENTLIGHT   12 ; luz ambiente

vs_1_0

dcl_position0 v0
dcl_position1 v3
dcl_normal0 v1
dcl_normal1 v4
dcl_texcoord0 v2

def c13, 0.0, 0.0, 0.0, 0.0

; interpola a posição entre um stram e outro
mul r1, v0, c[CV_BLEND_FACTORS].y
mad r1, v3, c[CV_BLEND_FACTORS].x, r1

; o mesmo para as normais
mul r2, v1, c[CV_BLEND_FACTORS].y
mad r2, v4, c[CV_BLEND_FACTORS].x, r2

; transforma para o clip space
m4x4 oPos, r1, c[CV_WORLDVIEWPROJ_0]
mov oT0.xy, v2

; transforma a normal no world space e renormaliza
m3x3 r0.xyz, r2, c[CV_WORLD_0]
dp3 r0.w, r0, r0
rsq r0.w, r0.w
mul r0, r0, r0.w

; transforma o vértice em world space
m4x4 r3.xyzw, r1, c[CV_WORLD_0]




add r2, c[CV_LIGHTPOS_0],-r3 ; r2 = vetor da luz = Lp - Vp
mov r7, r2

dp3 r2.w, r2, r2             ; normaliza r2
rsq r2.w, r2.w
mul r2, r2, r2.w

dp3 r10, r0, r2              ; r10 = normal * luz
mul r10, r10, c[CV_LIGHT0]   ; r10 *= cor da luz

dp3 r8.w, r7, r7                         ; quadrado da distâcia (Lp - Vp)
mul r4.w, c[CV_LIGHT0].w, c[CV_LIGHT0].w ; quadrado do alcance da luz

slt r7.w, r8.w, r4.w         ; dist < range ? 1 : 0
mul r10, r10, r7.w           ; se o vertice está fora do alcance
                             ; então sua cor = 0.0, senão, ele
                             ; permanece inalterado

rcp r7.x, r4.w               ; r7.x = dist / range
mul r7.w, r7.x, r8.w

add r7.w, c[CV_CONSTANTS].z, -r7.w       ; r7.w = 1 - r7.w
mul r10, r10, r7.w                       ; suaviza a passagem de intensidade
                                         ; da iluminação entre a posição da
                                         ; luz e seu alcance máximo

max r10, c13, r10            ; corrige a cor caso seja menor que 0
add oD0, r10, c[CV_AMBIENTLIGHT]

Download

Você pode fazer o download do programa pelo seguinte endereço: www.unidev...ShaderCompiler.zip, o código fonte está incluso dentro deste pacote ZIP.

Observações importantes

-Este programa não compila shaders escritos no formato FX, do framework do D3DX.
-Quaisquer versões de shader incompatíveis com o DirectX 8/9 não serão reconhecidos por este compilador.
-Não se esqueça que este compilador é baseado no DirectX 9. Portanto as instruções dcl_ são necessárias no inicio de cada vertex shader para definir os registradores de entrada/input.
